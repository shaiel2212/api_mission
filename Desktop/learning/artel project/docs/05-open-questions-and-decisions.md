## 3. פערים שזוהו בין מסמכים למציאות מימוש

1. **אינטגרציות "API"** במסמכי שרת/טכני — לפרק ל: מה ב־UI בלבד ומה דורש שרת.  
2. **Sequelize** — מפורט בשרת; לא בטכני — לא סתירה; רק לזכור לאחד ברפרנס ארכיטקטורה.  
3. **קוד הצלחה** — 200 בחוזה; לוודא שהקליינט לא מצפה ל־201.  
4. **CSRF** — תקף בעיקר עם cookies לסשן; ל־SPA JSON same-origin לרוב מספיק + CORS מדויק אם סאבדומיין.  
5. **Rate limit** — לפרט: לפי IP, סף לדקה, סביבות dev/staging.

## 4. צ'קליסט לפני ספרינט ראשון של פיתוח

- [ ] מבנה דף הבית (ארוך / מפוצל) נקבע ומתועד.  
- [ ] חוזה JSON לשגיאות (`error`, `code`, `message`) מוסכם בין קליינט לשרת.  
- [ ] מדיניות כפילויות לידים (חלון, שדה מזהה) מוגדרת.  
- [ ] מדיניות לוגים ו־PII מוגדרת.  
- [ ] משתני סביבה: DB, PORT, CORS origin, (אופציונלי) מפתחות הצפנה.  
- [ ] מסלול פריסה: אותו דומיין ל־SPA+API או סאבדומיין — משפיע על CORS.  
- [ ] תוכן ראשוני: רשימת לקוחות, ציטוטי המלצות, פרויקטים לדוגמה — זמינים או placeholders מאושרים.
