# תיעוד אפיון — אתר ארטל בניה ופיתוח

מסמכים אלה מאגדים את כל האפיונים שהועברו לפרויקט, בסדר מומלץ לקריאה וליישום.

## סדר עבודה מומלץ

1. **[01-brd.md](./01-brd.md)** — דרישות עסקיות (מקור האמת לתוכן ולמסרים).
2. **[02-client-specification.md](./02-client-specification.md)** — אפיון קליינט (React, routing, UI, ביצועים, נגישות).
3. **[03-server-specification.md](./03-server-specification.md)** — אפיון שרת (Express, MySQL, API לידים, אבטחה).
4. **[04-technical-specification.md](./04-technical-specification.md)** — אפיון טכני מאוחד (ארכיטקטורה, חוזה API, מודל נתונים).
5. **[05-open-questions-and-decisions.md](./05-open-questions-and-decisions.md)** — שאלות פתוחות, פערים, והמלצות טכניות לסגירה לפני פיתוח.
6. **[עיצוב / ייצוא UX Pilot](./design/README.md)** — מיפוי קבצי הסקיצה שנחלצו מה־ZIP.

## כללי שימוש בתהליך עבודה

- לפני ספרינט / מilestone: לעבור על **05** ולסמן V על כל סעיף שנסגר עם הלקוח או נקבע פנימית.
- לפני מסירה: לוודא שכל דף ב־**01** מכוסה ב־**02**–**04** (או מתועד כהחלטה מפורשת).
- שינוי דרישה: לעדכן קודם את **01**, ואז ליישר את **02**–**04** בהתאם.
